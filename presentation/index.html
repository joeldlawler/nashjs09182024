AWS 6 Pillars of Architecture

1. Operational Excellence

This pillar focuses on running and monitoring systems to deliver business value, and continually improving processes and procedures.

2. Security

The security pillar emphasizes protecting information, systems, and assets while delivering business value through risk assessments and mitigation strategies.

3. Reliability

This pillar focuses on ensuring a system can recover from infrastructure or service disruptions, dynamically acquire computing resources to meet demand, and mitigate disruptions.

4. Performance Efficiency

The performance efficiency pillar concentrates on using computing resources efficiently to meet system requirements and maintaining that efficiency as demand changes and technologies evolve.

5. Cost Optimization

This pillar focuses on avoiding unnecessary costs.

6. Sustainability

The sustainability pillar focuses on minimizing the environmental impacts of running cloud workloads.

Data Structures

Data structures are specialized formats for organizing and storing data. They provide a way to manage large amounts of data efficiently for various uses. Some key data structures include:

  • Arrays: Ordered collections of elements accessed by index
  • Linked Lists: Sequences of nodes where each node points to the next
  • Stacks: Last-in-first-out (LIFO) collections
  • Queues: First-in-first-out (FIFO) collections
  • Trees: Hierarchical structures with parent-child relationships
  • Graphs: Networks of interconnected nodes
  • Hash Tables: Key-value pair collections for fast lookups

The choice of data structure impacts how efficiently data can be accessed, searched, inserted, and deleted.

Algorithms

Algorithms are step-by-step procedures or formulas for solving problems and performing computations. They define the logical sequence of operations to be performed on data structures. Some important categories of algorithms include:

  • Searching: Finding specific items in a data set
  • Sorting: Arranging data in a particular order
  • Graph algorithms: Analyzing networks and relationships
  • Dynamic programming: Breaking problems into simpler subproblems
  • Divide and conquer: Recursively breaking problems into smaller ones

Algorithms are evaluated based on their efficiency, usually measured by time complexity (how runtime scales with input size) and space complexity (memory usage).
